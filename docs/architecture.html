Core Components

Cal Gateway

The central hub running on your Mac. A Node.js service that:

  • Receives messages from all channels
  • Routes model turns to Claude through your configured API endpoint
  • Executes tool calls for calendar, mail, files, search, and local workflows
  • Manages session state and memory
  • Sends responses back to the originating channel

Session Manager

Maintains conversation context across channels and sessions:

  • Shared session: channels can share one conversation
  • Session Bridge: automatic context preservation at token limits
  • Session Recovery: graceful recovery from malformed or corrupted state

Memory System

Five-layer persistent memory:

  • Session memory: current conversation context
  • Daily logs: markdown files per day
  • Long-term memory: curated facts via consolidation
  • Keyword index: full-text search
  • Semantic memory: local vector database for meaning-based recall

See Memory: The Five Layers for the full picture.

Channel Architecture

Terminal

Direct invocation of Claude Code with Cal's instructions, memory, and local tool access.

Web UI (PWA)

HTTP server and WebSocket connection for real-time browser messaging.

iMessage

Local polling through imsg to read incoming messages and send responses through Messages.

Telegram

Bot adapter that forwards messages into the same Cal runtime and returns responses remotely.

Self-Evolving Capabilities

Session Bridge

When approaching the context limit, Cal generates a handoff, writes active context, and starts fresh with that context loaded.

Startup Memory Refresh

Nightly consolidation can refresh a compact startup memory so new sessions begin with recent, high-signal context.

Memory Consolidation

Scheduled consolidation extracts durable facts, updates long-term memory, and refreshes search indexes.

Data Flow Example

When you send "What's on my calendar today?" via iMessage:

  1. iMessage channel detects a new message in the local Messages database
  2. Cal Gateway receives the message
  3. Session Manager adds it to conversation history
  4. Gateway sends context and message to Claude through your configured API endpoint
  5. Claude decides to call the calendar tool
  6. Gateway executes the local calendar command and returns the result
  7. Claude generates a human-readable response
  8. Gateway sends the response back through iMessage

Key Design Decisions

Local-first

Everything runs on your Mac. The model API is the main external dependency.

Shared context

Channels share memory so the conversation follows you.

Plain files

Memory is stored as markdown and JSON: readable, editable, deletable.

Continuity-first

Session Bridge and startup-memory refresh preserve useful context across session boundaries.
