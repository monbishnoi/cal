Privacy

Local-first, privacy-first. Your data stays yours.

The Core Principle

Cal is designed around a simple idea: your data never leaves your Mac unless you explicitly send it somewhere.

This is not just a feature. It is the architecture. Cal runs on your machine, stores memory locally, and reaches external services only when the configured workflow requires it.

What Stays Local

Your Conversations

Messages are stored in local session files. No cloud sync. No telemetry. No model training on your private data.

Your Memory

Cal's memory lives in plain markdown and JSON files. You can read, edit, move, or delete them anytime.

Your Calendar, Mail, Notes

Cal reads local apps through local APIs and command-line tools. Relevant snippets are sent to the model only when needed for your request.

Your Files

When Cal reads or edits files, it does so directly on your filesystem.

What Leaves Your Mac

Cal needs a model API to reason. Here is the default boundary:

WhatWhereWhy
Your messagesClaude API or your configured model endpointTo get responses
Relevant contextClaude API or your configured model endpointSo the model can answer with the needed context
Web searchesYour configured search providerOnly when you ask Cal to search the web

No analytics. No usage tracking. No external memory sync unless you configure one.

Model API Boundary

Cal can use Anthropic's Claude API directly or a compatible endpoint you configure. The gateway treats the model provider as a reasoning service, not as the source of truth.

  • Local source of truth: memory and state remain in your local files
  • Bring your own key: credentials live in your local environment/config
  • Explicit context: Cal sends the model the context needed for a request, not your entire filesystem

iMessage Privacy

  • Messages read locally: Cal reads the Messages database on your Mac
  • Identity separation: Cal can respond from a separate configured identity
  • No relay server: messages are not proxied through a custom service

You Are in Control

See everything Cal knows

Memory is stored as files you can inspect directly.

Delete anything, anytime

Want Cal to forget something? Delete or edit the relevant memory file.

Audit the code

Cal is open source. Read the code, understand what it does, and modify it for your environment.

What Cal Will Never Do by Design

  • Send your data to third parties for training
  • Store your conversations on a Cal-owned server
  • Track your usage for analytics
  • Share your information across users
  • Access data you have not explicitly granted access to
