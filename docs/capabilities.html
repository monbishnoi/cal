Google Workspace MCP

Google Workspace is available as an optional MCP server for document workflows. It uses Google's gws CLI locally, so authentication stays under your Google account and the gateway only exposes the approved Phase 1 tools.

ToolWhat It DoesBoundary
google_drive_search_docsSearches Google Docs in DriveRead-only
google_docs_readReads a Google Doc by IDRead-only
google_docs_createCreates a new Google DocRequires write confirmation
google_docs_batch_updateApplies non-destructive Docs API batch updatesRequires write confirmation

Setup

npm install -g @googleworkspace/cli
gws auth login

Memory: The Five Layers

Cal does not just dump files and search them. It actively builds five layers of memory, culminating in local semantic understanding.

LayerWhat It IsHow It Works
1. Session MemoryCurrent conversation contextLives in RAM and is shared across channels
2. Daily LogsRaw chronological recordMarkdown files written during sessions
3. Long-term MemoryCurated facts and preferencesConsolidation extracts durable signal from daily logs
4. Keyword IndexFull-text searchBM25-style indexing for direct lookup
5. Semantic MemoryAssociative understandingLocal embeddings create a vector database for meaning-based recall

What's Working in the Background

Session Bridge

When you approach token limits, Cal preserves active context so the next session can pick up where the work left off.

Session Recovery

If something goes wrong mid-conversation, Cal detects it and recovers without throwing away useful context.

Startup Memory

Cal can refresh a compact, ranked context file after consolidation so new sessions load the most useful current context.

Persistent Memory

Daily logs, curated memory, keyword indexing, and semantic indexing turn usage into compounding context.
